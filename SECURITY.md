# Security Policy

VibeAudit is security-focused software, so responsible disclosure matters.

## Reporting a vulnerability

Please do not report vulnerabilities in public issues.

Preferred reporting options:

1. Use GitHub's private vulnerability reporting / security advisory flow if available.
2. If private reporting is not available, contact the repository owner through their GitHub profile and include enough detail to reproduce the issue.

Please include:

- A concise description of the vulnerability
- Affected component or URL, if applicable
- Reproduction steps or proof of concept
- Potential impact
- Suggested fix, if you have one

## Response expectations

Maintainers aim to acknowledge valid reports within 7 days and provide a remediation plan or status update as soon as practical. Timelines may vary based on severity and maintainer availability.

## Scope

Security-sensitive areas include:

- Authentication and session management
- Repository access and scan permissions
- Secrets handling and data retention
- Scan worker sandboxing and command execution
- Report access controls
- Payment and billing flows
- Dependency supply chain

## Safe harbor

Good-faith research that avoids privacy violations, data destruction, service disruption, and unauthorized access to third-party systems is welcome. Please stop testing and report immediately if you encounter sensitive data.
